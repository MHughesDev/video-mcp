# Phase 11 — Integration Testing (Real Media)

## Status
In Progress — 55%

## Goal
Prove the pipeline actually works on real media with real FFmpeg. Today every
test monkeypatches FFmpeg/ffprobe, so no test has ever rendered, graded, or
validated a real frame. This phase introduces small golden-media fixtures and
opt-in end-to-end tests that exercise the true binaries. **This phase is the
primary MVP gate** (MVP criteria 2 and 8).

## Depends On
Phase 5 (Render), Phase 8 (Validation), and Phase 9 (Workflow) must exist to be
tested end-to-end. Touches Phases 2, 4, 6, 7 for their real-media verifications.

## Tools / Components Delivered
This phase delivers **tests and fixtures**, not MCP tools.

| Component | Description | Status | Notes |
|-----------|-------------|--------|-------|
| Monkeypatched integration suite | `test_workflow_integration.py` — 19 tests, no real media | ✓ Done | Good structural coverage; no real FFmpeg |
| Golden-media fixtures | Tiny real clips/audio/LUT checked in or generated | ✗ Missing | The core deliverable |
| Real-FFmpeg render test | Render a fixture to a playable, correctly-sized `.mp4` | ✗ Missing | MVP criterion 2 |
| Real end-to-end workflow test | `edit_video_from_prompt` on real media → all variants + OTIO | ✗ Missing | MVP criterion 8 |
| Validation true/false-positive tests | Real black/silent/frozen vs. good clips | ✗ Missing | Proves Phase 8 thresholds |
| `pytest` markers + skip logic | `@pytest.mark.realmedia`, auto-skip when binaries/fixtures absent | ✗ Missing | Keeps CI green without FFmpeg |

## Acceptance Criteria
- [ ] Tiny golden-media fixtures exist (a few seconds of 1080p video, a short steady-tempo music clip, one `.cube` LUT) — either committed (small) or generated by a fixture script.
- [ ] A real-FFmpeg test renders a fixture timeline to a **playable `.mp4`** and asserts duration/resolution via ffprobe.
- [ ] A real end-to-end `edit_video_from_prompt` test produces validated outputs for **all three platforms** + a valid `.otio`.
- [ ] Validation detectors are proven on real media: a genuinely black/silent/frozen render is **rejected**, a good one **passes**.
- [ ] `apply_lut` and at least one grading preset are verified to change color on a real frame.
- [ ] Beat detection is verified against the fixture's **known BPM** within ±2 BPM.
- [ ] Real-media tests are **marked and skip cleanly** when FFmpeg/librosa or fixtures are unavailable, so the default unit run stays green and fast.

## Implementation Tasks

1. **Fixture strategy** — decide committed vs. generated.
   Done-when: a documented approach (e.g. a `tests/fixtures/` dir with a
   sub-2MB clip, or a `conftest.py` that synthesizes a clip with FFmpeg
   `testsrc`/`sine` on demand). **Status: Not Started.** *Recommendation:
   generate with `testsrc`/`sine` to avoid committing binaries.*
2. **Real-media markers** — `@pytest.mark.realmedia` + skip-if-missing.
   Done-when: `pytest -m "not realmedia"` is the default fast path; the marker
   runs only when binaries+fixtures exist. **Status: Not Started.**
3. **Render smoke test** — real `render_project` on a fixture.
   Done-when: output exists, ffprobe confirms duration±tolerance and exact
   platform dimensions. **Status: Not Started.** (Closes the P5 gating task.)
4. **End-to-end workflow test** — real `edit_video_from_prompt`.
   Done-when: all three variants + OTIO produced and validated; delivery
   manifest matches reality. **Status: Not Started.** (Closes the P9 gating
   task.)
5. **Validation truth tests** — known-bad and known-good fixtures.
   Done-when: black/silent/frozen rejected; good passes. **Status: Not
   Started.** (Closes the P8 gating task.)
6. **Grade + beat verification** — real LUT/preset color change; known-BPM beat
   check. Done-when: assertions on real output. **Status: Not Started.** (Closes
   P4/P6/P7 gating tasks.)
7. **OTIO round-trip** — confirm the exported `.otio` opens in `otioview` or an
   NLE. Done-when: a test (or documented manual check) passes. **Status: Not
   Started.** (Closes the P3 gating task / MVP criterion 6.)

## Test Coverage Requirements
- The 19 existing monkeypatched integration tests stay as the structural layer.
- New real-media tests live under `tests/integration/` behind the `realmedia`
  marker.
- CI must run unit + monkeypatched integration on every push; real-media tests
  run where FFmpeg is available (see P12).

## Known Gaps
- **Everything real is missing.** The 55% reflects strong *structural* (mocked)
  integration coverage; the real-media half is entirely unbuilt.
- No fixture pipeline, no `realmedia` marker, no real render/validate/grade/beat
  assertions, no OTIO third-party round-trip check.

## Notes
- This is the highest-leverage remaining work: it simultaneously closes the
  gating tasks in Phases 2–9 and unlocks MVP criteria 2, 5, 6, and 8.
- Prefer **generated** fixtures (FFmpeg `testsrc`/`sine`, a synthesized `.cube`)
  over committed binaries to keep the repo small and the tests hermetic. Commit
  only what cannot be generated (e.g. a tiny real-music clip for beat realism, if
  needed).
- Generated synthetic media is enough for criteria 2 and 5; criterion 3 (beat
  sync on *real* music) may need one small real, license-clear audio fixture.
